1. Motive

The motive is a frame-local gate, not a blended layer. If the evader is visible and at least one chase strategy is enabled, the chaser chooses chase. Otherwise it chooses knowledge acquisition.

shouldChase = evaderLocation.visible && anyEnabled(evaderPredictionPursuit, lineOfSightPursuit) motive = shouldChase ? "chase" : "knowledgeAcquisition"

2. Strategy Proposal

Each active strategy proposes a common action payload. Inactive strategies still exist in the proposal object, but do not enter weighting.

Identity
id, active, pursuitSource
Weight
confidence, usually in the range 0..1
Intent
goalDirection, target candidate or pursuit point
Feasible path
actionPath and firstAction

3. Action

The final vehicle action is currently taken from action-path consensus. If there is no consensus path, the code falls back to spin or visible-bearing steering.

if actionPathConsensus.firstAction: forward = firstAction.forward reverse = firstAction.reverse steering = firstAction.steer

Chase Strategies

Evader prediction pursuit
Selects a pursuit point from the evader prediction path. Confidence comes from evaderPrediction.prediction.consensus. The feasible path steers toward the selected projected position.
Line of sight pursuit
Uses the currently visible evader bearing as a target direction. Confidence is fixed at 1. This is the direct visual fallback inside the chase motive.

Knowledge Acquisition Strategies

Map discovery
Scores known cells by frontier value, distance, and bearing. A frontier cell is a known cell next to unknown traversable space.
discoveryScore = frontier * 0.70 + distanceScore * 0.20 + bearingScore * 0.10
Map recency refresh
Scores known cells by age debt, distance, and bearing. It is explicitly disabled while map discovery has an active frontier target.
recencyScore = recencyDebt * 0.82 + distanceScore * 0.12 + bearingScore * 0.06
Spin
Generates a curved sweep path from the current heading. Confidence is fixed at 0.35. It is active only in the knowledge acquisition motive.

How Weighting Is Implemented

Direction consensus
Active proposals are converted to directional oscillator inputs using firstAction.predictedDirection when available, otherwise goalDirection. Kuramoto consensus produces peerConsensus and goalDirection.
peerSignal = { id: proposal.id, direction: firstAction.predictedDirection || goalDirection, confidence: proposal.confidence, weight: 1 }
Important current detail
The Kuramoto helper stores confidence, but its interaction and circular mean use weight. The chaser currently passes weight: 1, so proposal confidence does not weight the direction consensus.
Action-path consensus
The final action path uses proposal confidence directly. For each future frame, throttle and steering are averaged across active proposal paths, weighted by clamped proposal.confidence.
weight = clamp(proposal.confidence, 0, 1) mixedThrottle = sum(frame.throttle * weight) / sum(weight) mixedSteering = sum(frame.steer * weight) / sum(weight)

What Is Not Blended Yet

Motives do not currently compete in the same frame. The motive gate chooses chase or knowledge acquisition first, and only strategies inside that selected motive can contribute to proposal weighting.
current: motive gate -> strategies inside motive -> action not current: all motives -> all strategies -> action

Debug Payload Mental Model

The returned chaser action includes both the selected controls and the explanation payload. Debug views can inspect actionStrategies / actionPlan.proposals to see the motive signal, peer consensus, action-path consensus, local navigation placeholder, knowledge signal, and every strategy proposal.

return { forward, reverse, steering, pursuitPoint, movement, actionPath, chosenStrategy, actionStrategies: actionPlan.proposals, actionPlan }
